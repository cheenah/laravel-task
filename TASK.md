# Техническое задание

**Тестовое задание для Senior Fullstack Developer (Laravel + Vue/Blade)**  
**Срок выполнения:** до 3 (трёх) календарных дней

## Технологический стек (обязательный)

### Backend
- PHP 8.1+
- Laravel (актуальная LTS или последняя стабильная)
- MySQL или PostgreSQL
- Laravel Passport OAuth

### Frontend
- Vue.js **или**
- Blade

---

## Общая концепция системы

Необходимо реализовать веб-приложение управления заявками (Task / Request system).  
Система предназначена для создания заявок пользователями, их обработки менеджерами и администрирования.

---

## Пользователи и роли

### Роли пользователей:
- admin
- manager
- user

### Действия по ролям

| Действие              | admin | manager | user |
|------------------------|:----:|:------:|:---:|
| Просмотр всех заявок    | ✓    | ✓      | ✗   |
| Просмотр своих заявок   | ✓    | ✓      | ✓   |
| Создание заявки         | ✓    | ✓      | ✓   |
| Редактирование заявки   | ✓    | ✗      | ✓ (только new) |
| Смена статуса           | ✓    | ✓      | ✗   |
| Комментирование         | ✓    | ✓      | ✓   |

---

## Заявки (Tasks)

### Структура заявки:
- id
- title (string, required)
- description (text, optional)
- status (enum)
- author_id (user)
- created_at
- updated_at

### Статусы:
- new
- in_progress
- done
- cancelled

### Бизнес-ограничения:
- Пользователь может редактировать заявку только в статусе `new`
- Заявки в статусах `done` и `cancelled` нельзя редактировать
- Статус может менять только `manager` или `admin`
- Любая попытка нарушения логики должна корректно обрабатываться (403 / 422)

---

## Комментарии

- Комментарии привязаны к заявке
- Комментарии видят все пользователи, имеющие доступ к заявке
- Комментарии могут добавлять все роли
- Комментарии не редактируются и не удаляются

---

## API

### Минимальный набор эндпоинтов:

| Метод | URL | Назначение |
|-------|-----|------------|
| GET   | /api/tasks | Список заявок |
| POST  | /api/tasks | Создание заявки |
| GET   | /api/tasks/{id} | Просмотр заявки |
| PUT   | /api/tasks/{id} | Редактирование |
| PUT   | /api/tasks/{id}/status | Смена статуса |
| POST  | /api/tasks/{id}/comments | Добавление комментария |

### Требования к API:
- JSON-ответы
- HTTP-коды по стандарту REST
- Валидация входящих данных
- Авторизация

---
